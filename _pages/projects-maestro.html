Maestro: QoE-Aware Dynamic Resource Allocation in Wi-Fi Networks

Reinforcement learning resource manager that maximizes user QoE and QoE fairness without modifying clients, pairing telemetry-driven QoE estimation with flow-level access category and priority control.

Telemetry-only QoE estimation DDQN multi-agent RL Enterprise Wi-Fi 6 testbed

The challenge

QoS-tuned Wi-Fi policies miss user experience; static mappings (e.g., WMM) ignore changing load, and prior dynamic policies can require client-side hooks to see QoE.

Maestro approach

Formulates Wi-Fi resource allocation as a PO-MDP. LSTM QoE estimators (per application class) consume only controller telemetry; a DDQN multi-agent policy jointly selects access category (BE/VI/VO) and traffic priority per flow.

Evidence

Enterprise Wi-Fi 6 testbed (Aruba controller + AP, Real clients and servers) running video streaming, video conferencing, file transfer, and audio conferencing. Maestro lifts QoE up to 25× and fairness 78% over WMM, reaches up to 69% QoE over QFlow, and improves popular OTT apps by 12% on average (up to 29%).

Design highlights

Telemetry → QoE estimator

  • Uses only WLAN controller telemetry (257 features; 4-second sampling) to keep clients and apps untouched.
  • LSTM with three stacked layers (256/128/64 hidden units) trained for 50 epochs using MSE + Adam.
  • Separate estimators for video streaming, video conferencing, file transfer, and audio conferencing.

RL policy (DDQN, multi-agent)

  • One agent per application class; state combines estimated QoE and telemetry.
  • Action: assign access category (BE/VI/VO) + traffic priority (high/low) per flow—no static binding to app class.
  • Replay buffer + controlled exploration for sample efficiency in high-dimensional Wi-Fi state/action space.

Simulation environment

  • Built from a “knowledge base” of 402 hours of telemetry+QoE (362K data points) across 15 Wi-Fi configurations.
  • Starts RL agents from random states and evaluates actions using the knowledge base to avoid costly live training.
  • Maintains alignment between telemetry windows and QoE via 4-second intervals.

Controller integration path

  • WLAN controller flow classifier tags application class; Maestro ingests telemetry and estimates QoE.
  • Policy agent sends access category + priority to AP configuration handler; applied without client changes.
  • Targets elephant flows with sustained traffic.

Evaluation snapshot

QoE & fairness vs. WMM

  • Up to 25× higher QoE and 78% higher QoE fairness across mixed workloads in the Wi-Fi 6 testbed.
  • Popular OTT apps (Netflix, Zoom) gain 12% on average and up to 29% QoE improvement.

Against QFlow

  • Matches or exceeds QoE without requiring client-side instrumentation; up to 69% QoE lift reported.

Training efficiency

  • Simulation harness accelerates DDQN training using the knowledge base instead of repeated on-hardware trials.
  • Validation losses stabilize after ~30 epochs; residuals centered near zero on test data (per paper Figure 2).
